==========================================================
FULL SYSTEM ARCHITECTURE: MediCose (Smart Medication Tracker)
==========================================================

1. USER WORKFLOWS (Functional Layer)
------------------------------------
PATIENT: Login -> Profile -> Appointments -> AI Chat (Context-Aware) -> Pharmacy -> Order Tracking.
DOCTOR:  Login -> Professional Profile -> Patient Management -> Lab Report Review -> Digital Prescription.
ADMIN:   Login -> Analytics -> Smart Doctor-Patient Matching -> Pharmacy Order Management.

2. TECHNICAL STACK (Infrastructure Layer)
-----------------------------------------
- FRONTEND: React.js (Vite), Scoped CSS, Context API (Theme/Auth).
- BACKEND:  Spring Boot 3.4, Spring Security, JWT (Stateless Auth).
- AI:       Groq API (Llama 3.3 Model) with RAG (Retrieval-Augmented Generation).
- DATABASE: MySQL (Relational Storage).

3. DATABASE SCHEMA (SQL Layer)
------------------------------
- users: [id, username, email, password, doctor_id]
- user_profile: [id, age, gender, blood_group, symptoms, known_disease, profile_photo_url, user_id]
- doctors: [doctor_id, user_name, email, password]
- doctor_profile: [id, full_name, specialization, degree_name, experience, clinic_address, degree_photo_url]
- appointment: [id, date, time, reason, status, user_id, doctor_id]
- prescription: [id, date, diagnoses, medicines, note, next_follow_up, user_id, doctor_id]
- lab_report: [id, report_name, report_date, file_url, user_id]
- pharmacy_order: [id, medicines, total_amount, status, delivery_address, estimated_time, user_id]
- notification: [id, message, is_read, timestamp, user_id, doctor_id]
